## 项目概述

### 项目名称
TSSAIPlatform-frontend（AI 训练平台前端项目）

### 技术栈
- **框架**: UmiJS 4.x
- **UI 组件库**: Ant Design 5.x
- **Pro 组件**: @ant-design/pro-components
- **语言**: TypeScript
- **样式**: Less + antd-style
- **状态管理**: UmiJS 内置状态管理

## 原型集成说明

### 集成来源
**TSSAIPlatform-frontend-prototype** 为 M6 页面设计的独立 HTML 原型项目，用于快速预览和确认交互与布局。已完成向主项目 **TSSAIPlatform-frontend** 的集成。

### 已集成内容

| 原型内容 | 主项目对应 | 说明 |
|----------|------------|------|
| `assets/js/config.js` | `src/constants/platform.ts` | API、上传限制、模型/数据集/任务类型、分页等常量 |
| `task-create-v2.html` | `src/pages/task/create/index.tsx` | 发起训练：三步（选择/上传模型 → 选择/上传数据集 → 配置参数），参数支持「表单填写」与「上传训练代码」 |
| 模型上传类型与备注 | `src/pages/model/upload/index.tsx` | 模型类型（CV/NLP）、备注必填、支持 .pt/.pth/.onnx 等 |
| 数据集上传不要求类型 | `src/pages/dataset/upload/index.tsx` | 上传时不再强制选择类型，与原型一致 |

### 发起训练页流程（与原型一致）

1. **步骤 1：选择/上传模型**
   - 选择已有模型：从接口/列表选择
   - 上传新模型：填写名称、版本、类型、备注（必填）、选择文件，快速上传后自动选中

2. **步骤 2：选择/上传数据集**
   - 选择已有数据集 / 上传新数据集，流程同上

3. **步骤 3：配置参数**
   - **表单填写**：基础参数（epochs、batch_size、learning_rate、optimizer）+ 高级参数（JSON）
   - **上传训练代码**：上传 .py / .pyx / .ipynb / .txt，最大 50MB

### 服务与常量

- **常量**：`src/constants/platform.ts`（API_CONFIG、UPLOAD_CONFIG、MODEL_TYPES、TASK_STATUS 等）
- **服务**：`src/services/platform.ts`（fetchModelList、fetchDatasetList、创建任务等，可与后端 API 对接）

### 后续可做

- 将模型列表、数据集列表的请求统一改为使用 `src/services/platform.ts` 与 `API_CONFIG`
- 模型/数据集上传接口与后端联调（分片、进度等）
- 任务创建接口按「表单参数」与「上传训练代码」两种方式分别对接

## M6 任务范围

### 资产管理模块

#### 1. 模型管理
- ✅ **模型列表页** (`/model/list`)
  - 模型列表展示（名称、版本、类型、上传时间、大小）
  - 模型搜索和筛选
  - 模型删除功能
  - 模型详情跳转
  - 模型下载功能
  
- ✅ **模型上传页** (`/model/upload`)
  - 模型文件上传（支持 .pt/.pth/.onnx/.pb 等单文件及 .zip；千问等大模型由多文件组成，需打包为 .zip 后上传）
  - 模型信息填写（名称、版本、类型、备注等）
  - 上传进度显示
  
- ✅ **模型详情页** (`/model/detail/:id`)
  - 模型详细信息展示
  - 模型元数据查看
  - 模型版本历史

#### 2. 数据集管理
- ✅ **数据集列表页** (`/dataset/list`)
  - 数据集列表展示（名称、类型、上传时间、大小、文件数）
  - 数据集搜索和筛选
  - 数据集删除功能
  - 数据集详情跳转
  - 数据集下载功能
  
- ✅ **数据集上传页** (`/dataset/upload`)
  - 数据集文件/文件夹上传
  - 数据集信息填写（名称、类型等）
  - 上传进度显示

### 训练调度模块

#### 1. 发起训练
- ✅ **训练任务创建页** (`/task/create`)（已按原型集成）
  - **任务名称**：必填，页面顶部输入框，提交时随请求提交（表单/上传训练代码两种模式均包含）
  - 三步：选择/上传模型 → 选择/上传数据集 → 配置参数
  - 模型：选择已有 或 本页快速上传（名称、版本、类型、备注、文件）；支持 .pt/.pth/.onnx 等单文件及 .zip（千问等大模型）
  - 数据集：选择已有 或 本页快速上传
  - 参数配置二选一：**表单填写**（按模型类型 CV/NLP 区分）或 **上传训练代码**（.py/.pyx/.ipynb/.txt）
  - **表单填写**：CV 为 epochs(100–300)、batch、imgsz、lr0、optimizer(SGD)；NLP 为训练基础配置(num_epochs、batch_size、learning_rate、max_seq_length)与 LoRA 配置(lora_r、lora_dropout)；两种类型均支持高级参数(JSON 可选)
  - 参数验证与提交

#### 2. 任务列表
- ✅ **训练任务列表页** (`/task/list`)
  - 任务列表展示（名称、模型、数据集、创建时间、状态、进度）
  - 任务状态筛选（待执行、运行中、成功、失败）
  - 任务终止功能（运行中的任务）
  - 任务删除功能
  - 任务详情跳转

#### 3. 训练结果
- ✅ **训练结果详情页** (`/task/detail/:id`)
  - 任务基本信息展示
  - 训练指标图表展示（使用 ChartPanel 组件）
  - 训练结果文件列表
  - 结果文件下载功能

[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# 5. 设置为默认版本

npm --version

## 常见问题与解决方案

### 1. 依赖安装问题

#### 问题：npm install 失败或超时

**解决方案：**
```bash
# 使用国内镜像
npm install --registry=https://registry.npmmirror.com

# 清除缓存后重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

#### 问题：Node.js 版本不匹配

**解决方案：**
- 使用 nvm 管理 Node.js 版本
- 安装 Node.js 20.x LTS 版本

### 2. 启动问题

#### 问题：端口被占用

**解决方案：**
- UmiJS 会自动使用其他端口，查看终端输出
- 或手动指定端口：`PORT=3000 npm start`

#### 问题：启动报错

**解决方案：**
- 检查 Node.js 版本（需要 >= 20.0.0）
- 检查依赖是否完整安装
- 检查代码语法错误：`npm run tsc`

### 3. API 接口问题

#### 问题：接口请求失败（CORS 错误）

**解决方案：**
- 检查 `config/config.ts` 中的代理配置
- 确保后端服务已启动
- 检查后端 CORS 配置

#### 问题：接口返回 404

**解决方案：**
- 检查接口路径是否正确
- 检查后端路由配置
- 查看 Network 标签页确认请求 URL

### 4. 类型错误

#### 问题：TypeScript 类型检查失败

**解决方案：**
- 检查类型定义是否正确
- 使用 `as` 进行类型断言（谨慎使用）
- 更新 `src/typings.d.ts` 中的类型定义

### 5. 组件问题

#### 问题：ProTable 不显示数据

**解决方案：**
- 检查 `request` 函数返回值格式是否正确
- 确保返回 `{ data, success, total }` 格式
- 检查 `rowKey` 配置是否正确
