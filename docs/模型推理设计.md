# 模型推理模块 — 前端设计文档

> 版本：v1.6  
> 状态：设计稿，待评审后分步实现  
> 适用范围：TSSAIPlatform 前端 + 后端接口对齐

---

## 1. 文档目的

本文档定义 **模型推理** 模块的产品决议、页面结构（**含 CV / NLP / 多模态差异化交互**）、接口形态、进度与结果展示、Mock 规范。

**核心原则：**

- 推理与训练 **任务分离**，推理任务有独立 ID。
- **模型只从「可推理模型」接口选择**，不走模型管理页面的选择器。
- **输入分两种模式**：
  - **批量**：大数据 / zip → **数据集管理**上传，任务引用 `datasetVersionId`。
  - **单文件**：走 **推理专用上传接口**（**禁止**使用 `/files/upload` 调试接口）。
- **一条推理任务** = **一个可推理模型** + **一种输入** + **一份推理结果**（异步 Job 产出，含页内预览与/或下载 artifact）；换模型或换输入 = **新建任务**。
- **异步推理 + 进度条**；详情页 **页内预览结果**，下载为补充能力。
- **删除推理任务** = 删除任务记录 + **推理结果** + **单文件上传对象**（不持久化）；**批量数据集** 不随任务删，在数据集管理手动删（见 6.5）。

本文档 **不包含实现代码**。

## 2. 产品决议（最终版）

| # | 议题 | 决议 |
|---|------|------|
| 1 | 推理与训练 | 独立推理任务列表；不以 `experimentId` 为主键。 |
| 2 | 批量输入 | zip / 大批量 → **数据集管理**上传 → `datasetVersionId`；删任务 **不删** 数据集。 |
| 3 | 单文件输入 | **推理专用上传** `POST /inference/inputs/upload`（及必要时分片）；**不用** `/files/upload`。 |
| 4 | 模型来源 | **仅** `GET /inference/models`（可推理模型列表）；**不**在创建页调用模型管理列表 UI/API。 |
| 5 | 任务组成 | 一任务 = **一个** `inferenceModelId` + **一种**输入（`single`/`batch`）+ **一份**推理结果（`result` / `outputDownloadUrl` 等，异步产出）；多模型对比 = 多条任务。 |
| 6 | 执行方式 | 异步 Job；列表/详情 **自动刷新（5s）** + **手动刷新** + **进度条**；非终态可 **停止任务**（见 §6.6）；创建时可配 **推理参数** 与 **自定义推理脚本**（见 §6.3.5–6.3.6）。 |
| 7 | 删除任务 | 删 **推理记录 + 推理结果**；**单文件** input 一并删除（不持久化）；**批量** 数据集 **不删**，数据集管理手动删。 |
| 8 | 任务类型 UI | CV / NLP / **MULTIMODAL** 创建页与详情 **结果区** 交互不同（见第 6 节）。 |

## 5. 输入模式：`single` vs `batch`

| | 批量 `batch` | 单文件 `single` |
|--|--------------|-----------------|
| 适用 | zip、多图包、大批量文本集 | 单图、单文本文件、多模态「一图+一问」 |
| 数据来源 | 数据集管理 → `datasetVersionId` | 推理专用上传 → `inferenceInputId` |
| 上传接口 | 现有 `/dataset/upload/*` | **`/inference/inputs/upload`**（及分片，见 7.2.8） |
| 列表「数据」列 | 数据集 Tag（名称 + 大小 + 条数） | 输入文件行（文件名 + 可选大小） |
| 删任务时输入数据 | **不删** 数据集；数据集管理 **手动删** | **删** 已上传 input 对象（不持久化） |
| 换数据再推理 | 选同一数据集或另一数据集 | 需重新上传 |

**创建任务时二者互斥**：`inputMode` + `datasetVersionId` **或** `inferenceInputId`（NLP 单条纯文本可走 `text` 字段，见 6.3）。

## 6. 页面与路由

### 6.1 菜单与路由

```
模型推理
├── 推理任务列表    /inference/list
├── 创建推理任务    /inference/create        hideInMenu
└── 推理任务详情    /inference/detail/:id    hideInMenu
```

### 6.2 列表页 `/inference/list`

统计（全部 / 运行中 / 已完成 / 失败）、搜索、**自动刷新（5s）**、**手动刷新**。

**列表不展示「输入方式」列**；`inputMode` 仍存在于接口与详情页，仅不在表格中单独成列。

#### 6.2.4 刷新与停止（列表）

| 能力 | 位置 | 行为 |
|------|------|------|
| **自动刷新** | ProTable `polling={5000}` | 每 5s 重新请求 list；`request` 内同步刷新 stats |
| **手动刷新** | 页头 `extra`「刷新」按钮（`ReloadOutlined`） | 调用 `actionRef.reload()` + `loadStats()`；不关闭自动轮询 |
| **停止任务** | 操作列（仅 `pending` / `queued` / `running`） | `Popconfirm` → `stopInferenceTask(id)` → 刷新列表与 stats |

**停止** 与 **删除** 区分：停止保留任务记录与已有结果片段，仅终止后续执行；删除见 §6.5。

**表格列**

| 列名 | 字段 | 说明 |
|------|------|------|
| 推理任务 ID | `id` | 见 **6.2.1**；等宽字体 |
| 任务名称 | `name` | |
| 任务类型 | `taskType` | CV / NLP / MULTIMODAL |
| 模型 | `modelDisplayName` | 见 **6.2.2**；可推理模型 Tag |
| 数据 | — | 见 **6.2.3**；批量=数据集块，单文件=输入文件行 |
| 推理状态 | `status` | |
| 进度 | `progress` | 运行中：迷你 `Progress` 或百分比 |
| 提交时间 | `createdAt` | |
| 操作 | — | 详情、停止（非终态，§6.6）、删除（`Popconfirm` 文案见 §6.5.3） |

#### 6.2.1 推理任务 ID（自动生成）

**格式**：`INF-{YYYYMMDD}-{SEQ}`

| 段 | 规则 | 示例 |
|----|------|------|
| 前缀 | 固定 `INF` | `INF` |
| 日期 | 任务 **创建日** `YYYYMMDD` | `20260628` |
| 序号 | 当日递增，**3 位**左补零 | `005` |

**完整示例**：`INF-20260628-005`

- **后端**在 `POST /inference/tasks` 创建时生成，作为唯一主键；前端 **只读展示**，不在创建表单填写。
- Mock 列表/详情中的 `id` 统一使用该格式（勿再用 `inf-cv-single-001` 这类 slug）。

**列表样式**：等宽字体（`font-family: monospace`），浅灰字色，深色背景下偏「系统编号」观感。

#### 6.2.2 模型列 — Model Tag

单行 **圆角 Tag**，左侧图标 + 模型名，**不**用纯文本列。

| 元素 | 说明 |
|------|------|
| 图标 | 芯片 / 立方体（表示模型） |
| 文案 | `modelDisplayName`，如 `ResNet-50 v2.3`、`Llama-3-8B-Instruct` |
| 样式 | 深色底 + 细边框 + 主题色字（CV 可偏青绿，NLP/多模态可偏蓝；与任务类型 Tag 色系可区分） |

组件：`InferenceModelTag`（`modelDisplayName` + 可选 `taskType` 配色）。

数据来源：`GET /inference/models` 选中项的 `displayName`，列表接口冗余 `modelDisplayName` 便于渲染。

#### 6.2.3 数据列 — 批量 vs 单文件

**批量 `batch`** — 数据集块（两行，非 Tag 单列）：

```
[数据库图标]  ImageNet-Val-2026
              6.4 GB · 50,000
```

| 字段 | 说明 |
|------|------|
| 主行 | `datasetName` 或 `datasetDisplayName` |
| 次行 | `formatBytes(datasetSizeBytes)` · `formatNumber(datasetItemCount)` |

**单文件 `single`** — 输入文件行（紧凑一行或两行，**不用**数据集图标语义）：

```
[文件图标]  photo.jpg
            2.1 MB          // 可选
```

| 字段 | 说明 |
|------|------|
| 主行 | `inputFileName` 或 `inputDisplayName` |
| 次行 | `inputSizeBytes`（有则展示） |

NLP 纯文本粘贴（无上传文件）时，主行可展示 `text` 摘要（如前 30 字 + `…`）。

组件：`InferenceDatasetCell`（批量）、`InferenceInputFileCell`（单文件）；列表页按 `inputMode` 二选一渲染。

### 6.3 创建页 `/inference/create` — **分任务类型不同交互**

**公共步骤（所有类型）：**

1. 任务名称  
2. **选择可推理模型**（`GET /inference/models`，选中后确定 `taskType` 与表单项）  
3. **输入方式**：单文件 / 批量（Radio）  
4. 备注（可选）

**选中模型后，按 `taskType` 渲染不同输入区（不共用一套 UI）：**

#### CV

| 输入方式 | 交互 |
|----------|------|
| **单文件** | 图片上传（拖拽）→ 调 `POST /inference/inputs/upload` → 得到 `inferenceInputId`；浏览器内预览缩略图 |
| **批量** | 选择 `datasetVersionId`（数据集列表 + 链到 `/dataset/upload` 上传 zip） |

#### NLP

| 输入方式 | 交互 |
|----------|------|
| **单文件** | 二选一（子 Tab）：**粘贴文本**（`text` 字段，不上传） / **上传 .txt 等**（`inferenceInputId`） |
| **批量** | 选择 `datasetVersionId`（文本集 zip / jsonl 等在数据集管理上传） |

#### MULTIMODAL

| 输入方式 | 交互 |
|----------|------|
| **单文件** | 图片上传（`inferenceInputId`）+ **Prompt 文本框**（`prompt` 字段，创建任务时一并提交） |
| **批量** | 选择 `datasetVersionId`（图文对 zip 等） |

**提交 body 见 7.2.3。**

#### 6.3.5 推理参数（任务级，必显区域）

对齐 SageMaker Batch Transform 环境变量 / HuggingFace Pipeline kwargs / OpenAI Completion 参数：**用户在创建任务时调整**，详情页 **只读展示**；任务创建后 **不可修改**（改参数 = 新建任务）。

**入口**：创建页 `/inference/create`，在「输入区」与「备注」之间，卡片 **「推理参数」**（默认展开，非 Collapse 隐藏）。

**交互**

| 项 | 规则 |
|----|------|
| 默认值 | 选中可推理模型后，合并 **平台默认** + 模型 `defaultInferenceParams`（`GET /inference/models` 可选字段） |
| 切换模型 | 重置为该模型对应 `taskType` 的默认参数 |
| 切换 taskType | 表单项随类型切换（CV / NLP / MULTIMODAL 字段不同） |
| 校验 | 数值范围见 §7.2.10；非法值阻止提交 |
| 提交 | `POST /inference/tasks` body 字段 `inferenceParams: Record<string, string \| number>` |

**按 taskType 的参数字段（v1）**

| 作用域 | 字段 | 类型 | 默认 | 说明 |
|--------|------|------|------|------|
| **公共** | `device` | select | `cpu` | `cpu` / `cuda` / `cuda:0` |
| **公共** | `batch_size` | number | `8` | 批量 Job 内部 micro-batch；1–128 |
| **CV** | `confidence` | number | `0.25` | 检测置信度阈值；0–1 |
| **CV** | `iou_threshold` | number | `0.45` | NMS IoU 阈值；0–1 |
| **CV** | `img_size` | number | `640` | 输入缩放边长 |
| **CV** | `max_detections` | number | `100` | 单图最大检测框数 |
| **NLP** | `max_tokens` | number | `512` | 最大生成 token |
| **NLP** | `temperature` | number | `0.7` | 采样温度；0–2 |
| **NLP** | `top_p` | number | `0.9` | nucleus；0–1 |
| **NLP** | `top_k` | number | `50` | top-k 采样 |
| **MULTIMODAL** | 同 NLP | — | — | 生成类参数与 NLP 共用 schema |

**组件**：`InferenceParamsForm.tsx`（读 `inferenceParamSchema.ts` 渲染 Form.Item）。

**详情页**：基本信息下增 **「推理参数」** `Descriptions` 或 JSON 只读块（`InferenceParamsDisplay`）。

**Schema 接口（可选）**：`GET /inference/param-schema?taskType=CV` 返回字段定义；前端 v1 可 **constants 写死**，接口供后端对齐。

#### 6.3.6 高级选项 — 自定义推理脚本

对齐 SageMaker **Inference Entry Point**（`inference.py` + `model_fn`/`input_fn`/`predict_fn`）、Azure ML **scoring script**、BentoML **自定义 service**：允许用户上传 **Python 推理 handler**，覆盖平台默认后处理/前后处理逻辑。

**入口**：创建页 **「高级选项」** `Collapse`（默认 **收起**）；与「推理参数」分离，避免普通用户误触。

**交互**

| 项 | 规则 |
|----|------|
| 开关 | `useCustomScript` Switch；默认 `false`（使用平台默认推理 handler） |
| 上传 | 开启后 **必传** `.py` 单文件；`POST /inference/scripts/upload` → `customScriptId` |
| 入口函数 | `scriptEntryPoint` 文本框，默认 `inference_handler`（对应脚本内 callable 名） |
| 大小限制 | ≤ 1MB；仅 `.py` |
| 说明 | `Alert`：脚本需实现约定签名 `inference_handler(context)` 或 SageMaker 风格四函数；详见 §7.2.11 |
| 提交 | body：`useCustomScript`, `customScriptId`, `scriptEntryPoint` |
| 详情 | 展示「推理方式：平台默认 / 自定义脚本」+ 文件名 + entryPoint |

**生命周期**（与单文件 input 一致，**不持久化**）：

- 存储路径：`inference/scripts/{customScriptId}/handler.py`
- **删任务**时若存在 `customScriptId`，**同步删除**脚本对象（`scriptDeleted: true`）
- **不**做跨任务复用、不做独立脚本管理页

**组件**：`InferenceAdvancedForm.tsx`（Collapse + Switch + Upload + entryPoint）。

**禁止**：高级选项 **不得** 走 `/files/upload`；须走 `/inference/scripts/upload`。

### 6.4 详情页 `/inference/detail/:id` — **进度 + 分类型结果区**

**公共区域**

| 区域 | 内容 |
|------|------|
| 基本信息 | ID、名称、类型、输入方式、模型、数据摘要、**推理参数**（§6.3.5）、**推理方式/自定义脚本**（§6.3.6）、时间 |
| **推理进度** | 见 6.4.1 |
| 失败 | `errorMessage` |
| 操作 | 返回、**刷新**、**停止任务**（非终态，§6.6）、删除（`Popconfirm` 文案见 §6.5.3） |

**结果区域按 `taskType` 切换不同组件（见 6.4.2）**

非终态：轮询 `GET /inference/tasks/:id`（5s）；进入终态（`success` / `failed` / `stopped`）后停止轮询。

#### 6.4.3 刷新与停止（详情）

| 能力 | 位置 | 行为 |
|------|------|------|
| **自动刷新** | `useEffect` + `setInterval(5000)` | 仅非终态任务轮询 `loadDetail()` |
| **手动刷新** | 页头 `extra`「刷新」按钮 | 立即 `loadDetail()`；不中断自动轮询 |
| **自动刷新提示** | 基本信息「状态」项旁 | 非终态展示 Tag：`自动刷新 · 5s`（与训练任务详情一致） |
| **停止任务** | 页头 `extra`（非终态显示） | `Popconfirm` → `stopInferenceTask(id)` → 刷新详情；成功后展示 `stopped` 状态（后续 UI 见 **§6.6.4**） |

#### 6.4.1 推理进度条（必须）

详情页 **运行中** 展示 `Progress`（antd）；**已停止** 时 **仍展示**，进度冻结（见 §6.6.4）。

| 字段 | UI |
|------|-----|
| `progress` | 0–100，主进度条 |
| `progressMessage` | 文案，如「正在推理第 3/10 张」 |
| `processedCount` / `totalCount` | 批量时显示「已处理 / 总数」 |

列表页运行中任务可选展示迷你进度（同一 `progress` 字段）。

**Mock 需模拟**：running 任务 `progress` 随时间递增（前端 mock 或静态 45/60 等）。

#### 6.4.2 推理结果如何看见（**不是只能下载**）

原则：**页内可直接阅读/预览**；完整批量结果提供 **下载** 作为补充。

| taskType | inputMode | 页内展示 | 下载 |
|----------|-----------|----------|------|
| **CV** | single | 输入图预览；**检测列表**（label、score、bbox）；若有 `annotatedImageUrl` 展示 **框图** | 可选 JSON |
| **CV** | batch | 汇总：总数、成功率；**结果表格**预览前 N 条；`processedCount/totalCount` | **`outputDownloadUrl`**（jsonl/zip） |
| **NLP** | single（文本） | **文本区**展示 `generatedText` / 分类 `label+score` / 实体列表 | 可选 txt/json |
| **NLP** | batch | 汇总 + **结果表格**前 N 条（文件名、label、摘要） | **下载完整 jsonl** |
| **MULTIMODAL** | single | 输入图 + **回答** `answer` / `generatedText` | 可选 |
| **MULTIMODAL** | batch | 汇总 + 表格预览 | **下载** |

**结论**：单文件以 **详情页内嵌只读展示** 为主；批量以 **摘要 + 抽样表格 + 下载全量** 为主。`outputDownloadUrl` 为批量或大结果必备，但不替代页内预览。

**组件划分（实现时）**

```
pages/inference/detail/[id].tsx          # 壳 + 进度 + 公共信息
components/results/CvResultPanel.tsx
components/results/NlpResultPanel.tsx
components/results/MultimodalResultPanel.tsx
components/results/BatchResultPreview.tsx   # 批量共用表格预览
```

### 6.5 删除推理任务

```
DELETE /inference/tasks/:id
```

#### 6.5.1 按输入模式区分

| | 单文件 `single` | 批量 `batch` |
|--|-----------------|--------------|
| 推理任务记录 | ✅ 删除 | ✅ 删除 |
| 推理结果 `inference/outputs/{taskId}/` | ✅ 删除 | ✅ 删除 |
| 输入数据 | ✅ **删除** 已上传的 `inferenceInputId` 对象（MinIO 等） | ❌ **不删** `datasetVersionId` 对应数据集 |
| 持久化 | **不持久化**；删任务即删 input，无需 GC / 输入管理 | 数据集生命周期 **独立**；在 **数据集管理** 手动删除 |

**NLP 纯文本**（`text` 字段，无 `inferenceInputId`）：删任务仅删 DB 记录与结果，无对象存储可删。

#### 6.5.2 汇总表

| 删除 | 不删除 / 手动处理 |
|------|-------------------|
| 推理任务 DB 记录 | 批量任务的 `datasetVersionId` 及数据集 MinIO 对象 |
| `inference/outputs/{taskId}/` 下推理 **结果** | 批量数据集 → **数据集管理** 手动删 |
| 单文件 `inference/inputs/{inferenceInputId}/` 下 **上传对象** | |
| 任务引用的 `inference/scripts/{customScriptId}/` **自定义脚本** | |

**前端提示（删除确认）**：见 **§6.5.3**（按 `inputMode` 区分文案；NLP 纯文本单独说明）。

#### 6.5.3 前端删除交互（列表 / 详情）

**统一入口**

| 页面 | 组件 | 调用 |
|------|------|------|
| 列表 `/inference/list` | `Popconfirm` + 删除按钮 | `deleteInferenceTask(id)` |
| 详情 `/inference/detail/:id` | `Popconfirm` + 「删除任务」 | 同上，成功后 `history.push('/inference/list')` |

**确认框文案（按 `inputMode` + 是否有 `inferenceInputId`）**

| 场景 | `title` | `description` |
|------|---------|---------------|
| `batch` | 确定删除该推理任务？ | 将删除任务记录及推理结果。**不会删除**关联的数据集，请在数据集管理中手动删除。 |
| `single` 且存在 `inferenceInputId` | 确定删除该推理任务？ | 将删除任务记录、推理结果及**已上传的输入文件**（不可恢复）。 |
| `single` 且仅 `text`（NLP 粘贴，无 upload） | 确定删除该推理任务？ | 将删除任务记录及推理结果。 |

列表页删除时若仅有 `InferenceTaskListItem`，可据 `inputMode` 判断：`batch` 用批量文案；`single` 默认用「含 input 文件」文案（与多数 CV/多模态/上传 txt 一致）。NLP 纯文本任务在列表项上无 `inferenceInputId` 字段时，可用 `inputDisplayName` 是否为长文本摘要 heuristic，或列表接口冗余 `hasInferenceInput?: boolean`（可选字段，见 §7.2.2）。

**成功提示**

```ts
// 统一 success message 模板（services 或 page 内）
message.success(
  res.data.inputDeleted
    ? '已删除推理任务、结果及输入文件'
    : '已删除推理任务及结果',
);
```

**失败**：展示 `errorMessage`；不刷新列表/不跳转。

**实现建议**

- 抽取 `getInferenceDeleteConfirm(record)` → `{ title, description }`（`src/pages/inference/utils/deleteConfirm.ts` 或 `constants/inference/deleteConfirm.ts`）。
- 列表、详情 **共用** 该函数，避免文案分叉。

#### 6.5.4 类型与 Service 约定

```ts
type InferenceTaskDeleteResult = {
  id: string;
  deleted: boolean;
  resultsDeleted: boolean;
  /** single 且存在 inferenceInputId 并已删对象时为 true */
  inputDeleted?: boolean;
};
```

`deleteInferenceTask(id)` 返回 `{ success, data: InferenceTaskDeleteResult }`；Mock 与真实后端 **字段一致**。

### 6.6 停止推理任务

```
POST /inference/tasks/:id/stop
```

#### 6.6.1 语义

| 项 | 说明 |
|----|------|
| 可停止状态 | `pending` \| `queued` \| `running` |
| 终态 | 成功后 `status` → **`stopped`**；写入 `finishedAt`；`progress` 冻结在当前值 |
| 保留内容 | 任务记录、输入引用（`datasetVersionId` / `inferenceInputId` / `text`）、**已产生的部分结果**（若有） |
| 不删除 | 与 §6.5 **删除** 不同：停止 **不删** 任务、不删 input、不删数据集 |
| 提示字段 | `errorMessage` 建议为「任务已手动停止」（前端亦可用固定文案） |

#### 6.6.2 前端交互（列表 / 详情）

**统一入口**

| 页面 | 组件 | 调用 |
|------|------|------|
| 列表 `/inference/list` | `Popconfirm` + 「停止」链接按钮 | `stopInferenceTask(id)` |
| 详情 `/inference/detail/:id` | `Popconfirm` + 「停止任务」按钮 | 同上，成功后留在详情页并刷新 |

**确认框文案**

| 字段 | 文案 |
|------|------|
| `title` | 确定停止该推理任务？ |
| `description` | 停止后任务将不再继续执行，已产生的部分结果可能不完整。 |

**成功提示**：`message.success('推理任务已停止')`；列表需 `reload` + `loadStats`。

**失败**：展示接口 `errorMessage`（如任务不存在、已是终态）。

**实现建议**

- 抽取 `getInferenceStopConfirm()` → `{ title, description }`（与 `getInferenceDeleteConfirm` 同目录）。
- 抽取 `isActiveInferenceStatus(status)`：`pending` \| `queued` \| `running`（`constants/inference/constants.ts`）。

#### 6.6.3 类型与 Service 约定

```ts
type InferenceTaskStatus =
  | 'pending'
  | 'queued'
  | 'running'
  | 'success'
  | 'failed'
  | 'stopped';   // v1.4 新增：用户手动停止

// stopInferenceTask(id) → { success, data: InferenceTaskDetail }
```

**统计卡片**：`stats.failed` **含** `status === 'failed'` 与 `status === 'stopped'`（「失败」卡片展示合并计数；列表/详情 Tag 仍区分「失败」与「已停止」）。

#### 6.6.4 停止后的任务处理（对齐业内 Batch Job 惯例）

**原则**（与 OpenAI Batch、Gemini Batch、SageMaker Job 等一致）：

- 停止 = **终止后续执行** + **保留 Job 记录与已完成产出**；
- **不支持** 在同一 Job ID 上 Resume / 继续执行；
- 若要处理剩余数据 → **新建推理任务**（v1 不提供「一键复制配置」，用户手动重新创建）。

**停止后用户可执行的操作**

| 操作 | 说明 |
|------|------|
| 查看详情 | 终态；不再自动轮询；可手动刷新 |
| 查看冻结进度 | 展示停止时的 `progress`、`processedCount` / `totalCount` |
| 查看 / 下载部分结果 | 若后端已写入 `result` / `outputDownloadUrl`，页内预览 + 下载（批量常见） |
| 删除任务 | 同 §6.5；与停止独立 |
| 重新推理 | 跳转 `/inference/create` 新建任务 |

**详情页 UI 规则（`status === 'stopped'`）**

| 区域 | 行为 |
|------|------|
| 页头操作 | 仅「刷新」「删除任务」；**不显示**「停止任务」 |
| 推理进度 | **仍展示**；Progress **冻结**于停止时百分比；卡片标题可为「推理进度（已停止）」；`progressMessage` 建议含「任务已停止于 …」 |
| 状态提示 | warning `Alert`：`errorMessage` 或「任务已手动停止」 |
| 推理结果 — **有部分结果** | 按 `taskType` 复用 ResultPanel；结果区上方 info `Alert`：「以下为停止前已完成的推理结果，可能不完整。若要处理剩余数据，请新建推理任务。」 |
| 推理结果 — **无结果** | info `Alert`：「暂无可用推理结果。可删除该任务，或重新创建推理任务。」（不渲染空 ResultPanel） |

**判定「有部分结果」**（前端 helper，实现时可抽 `hasInferenceResult(detail)`）：

`detail.result` 存在且至少满足其一：`predictions[]`、`annotatedImageUrl`、`label`、`generatedText`、`answer`、`entities[]`、`previewItems[]`、`outputDownloadUrl`、`summary`。

**后端约定**

| 项 | 说明 |
|----|------|
| stop 响应 | 返回更新后的 `InferenceTaskDetail`（含冻结进度） |
| 部分产出 | 批量 Job 停止前若已写出 output，应填充 `result.previewItems` / `outputDownloadUrl`；单文件若在停止前已完成推理，可填充 `result` |
| 不自动删 | 停止 **不** 触发删除 Job / input / 数据集 |
| 无 Resume | v1 **不提供** `POST .../resume` |

**实现建议**

- `InferenceProgress` 增加 `frozen?: boolean`：stopped 时进度条灰色、无动画。
- `pages/inference/detail/[id].tsx`：`renderResultPanel` 对 `stopped` 分支按上表处理。

### 8.6 Mock 删除行为（对齐 §6.5）

**函数**：`mockDeleteInferenceTask(id: string)`（`mockData.ts`）

**内存 Store 需维护**

| Store | 用途 |
|-------|------|
| `taskListStore` | 列表项 |
| `taskDetailStore` | 详情 |
| `inputStore` | `inferenceInputId → { objectName, previewUrl? }`（upload 时写入，delete 时移除） |

**删除步骤（Mock 模拟后端）**

1. 从 `taskDetailStore[id]` 或 `taskListStore` 读取 `inputMode`、`inferenceInputId`（详情优先）。
2. 从 `taskListStore`、`taskDetailStore` 移除该任务。
3. 若 `inputMode === 'single'` 且存在 `inferenceInputId`：
   - 从 `inputStore` 删除对应项；
   - 若曾 `URL.createObjectURL`，Mock 可忽略 revoke（真实后端删 MinIO）。
   - 设 `inputDeleted: true`。
4. 否则 `inputDeleted: false`（含 `batch`、NLP 纯 `text`）。
5. 始终 `resultsDeleted: true`、`deleted: true`（Mock 无独立 output 文件表，语义上结果随任务删）。

**Mock 响应示例 — 单文件 CV**

```json
{
  "success": true,
  "data": {
    "id": "INF-20260628-002",
    "deleted": true,
    "resultsDeleted": true,
    "inputDeleted": true
  }
}
```

**Mock 响应示例 — 批量**

```json
{
  "success": true,
  "data": {
    "id": "INF-20260628-001",
    "deleted": true,
    "resultsDeleted": true,
    "inputDeleted": false
  }
}
```

**Mock 列表项**：single 且经 upload 创建的任务设 `hasInferenceInput: true`；NLP 纯文本任务设 `hasInferenceInput: false`，供列表 `Popconfirm` 文案（§6.5.3）。

**注意**：Mock **不得** 从全局 `MOCK_DATASETS` / 数据集 Store 删任何数据。

### 8.7 Mock 停止行为（对齐 §6.6）

**函数**：`mockStopInferenceTask(id: string)`（`mockData.ts`）

1. 从 `taskDetailStore[id]` 读取任务；不存在则 throw。
2. 若 `status` 非 `pending` / `queued` / `running`，throw「任务已结束，无法停止」。
3. 更新 `taskDetailStore` 与 `taskListStore` 对应项：
   - `status: 'stopped'`
   - `finishedAt: now`
   - `errorMessage: '任务已手动停止'`
   - `progress` / `processedCount` 保持当前值；`progressMessage` 更新为「任务已停止于 {progress}%」类文案
4. **部分结果**（对齐 §6.6.4）：若 `inputMode === 'batch'` 且 `processedCount > 0`，写入 `result`：
   - `summary`: `{ total, success: processedCount, failed: 0 }`（`total` 取 `totalCount`）
   - `previewItems`: 抽样 2–3 条 Mock 预览行
   - `outputDownloadUrl`: `/api/inference/outputs/{id}/partial-results.jsonl`
5. 返回 `{ success: true, data: InferenceTaskDetail }`。
6. **不** 修改 `inputStore`、**不** 删数据集。

**stats**：`computeStats` 中 `failed` 计数含 `failed` + `stopped`。

## 11. 明确不做（v1）

- 创建页使用 **模型管理** 列表或跳转 `/model/list` 选模型。
- 推理模块调用 **`/files/upload`**。
- 一条任务多个模型。
- 删除推理任务时删除 **批量任务引用的数据集**（数据集仅在数据集管理手动删除）。
- 单文件 input **持久化**或删任务后 **保留** upload 对象（与 §6.5 相反）。
- 结果 **仅** 提供下载、无任何页内展示。
- 已停止任务 **Resume / 继续执行**（同一 Job ID）。
- 停止后 **自动删除** Job 记录。
