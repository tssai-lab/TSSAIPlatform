## 2. 项目概述

### 2.1 系统定位

本系统为 **AI 模型训练与管理平台** 的 Web 前端，面向企业内模型、数据集、训练任务及算力资源的统一管理，提供：

- 用户认证与分级权限控制；
- 模型与数据集的上传、管理与预览；
- 训练任务发起、监控、结果查看与对比分析；
- GPU 算力资源监控与排队调度可视化（超级管理员）；
- 系统用户、日志与配置管理（管理员）。

### 2.2 交付形态

| 交付项 | 说明 |
|--------|------|
| 源代码 | 完整 React 前端工程，含页面、组件、服务层与配置 |
| 构建产物 | 执行 `npm run build` 后生成 `dist/` 静态资源 |
| 运行脚本 | `scripts/serve-dist.cjs` 支持生产环境静态托管 + API 反向代理 |
| 配套文档 | 本文档、`README.md`、`DEPLOY_SERVER.md`、算力监控 API 文档等 |

### 2.3 技术栈

| 类别 | 技术 | 版本要求 |
|------|------|----------|
| 运行时 | Node.js | >= 20.0.0 |
| 包管理 | npm / yarn | npm >= 9 |
| 框架 | UmiJS Max | 4.x |
| UI 库 | Ant Design | 5.x |
| 企业组件 | @ant-design/pro-components | 2.x |
| 视图层 | React | 19.x |
| 语言 | TypeScript + JavaScript | 混用 |
| 图表 | ECharts、@ant-design/plots | 训练指标与资源趋势 |
| 3D 预览 | Three.js + @react-three/fiber | 点云等可视化 |
| 代码规范 | Biome、Commitlint、Husky | 提交前检查 |

## 3. 功能模块说明

### 3.1 模块总览

| 模块 | 路由前缀 | 主要功能 | 访问角色 |
|------|----------|----------|----------|
| 用户认证 | `/user/*` | 登录、注册、找回/重置密码 | 公开 |
| 首页 | `/dashboard` | 平台概览 | 登录用户 |
| 模型管理 | `/model/*` | 列表、上传、详情、代码预览 | 登录用户 |
| 数据集管理 | `/dataset/*` | 列表、上传、详情、点云查看 | 登录用户 |
| 训练调度 | `/task/*` | 任务列表、发起训练、详情、对比分析 | 登录用户 |
| 算力资源监控 | `/task/resourceMonitor/*` | 集群汇总、服务器卡片、详情、排队管理 | **仅超级管理员** |
| 系统管理 | `/system/*` | 用户、管理员、日志、系统配置 | 管理员（分级） |
| 个人中心 | `/account/*` | 我的操作日志 | 登录用户 |

### 3.2 训练调度子模块

| 页面 | 路径 | 说明 |
|------|------|------|
| 任务列表 | `/task/list` | 查看全部训练任务及状态 |
| 发起训练 | `/task/create` | 选择模型/代码/数据集并提交训练 |
| 任务详情 | `/task/detail/:id` | 任务信息、MLflow 训练指标、结果文件 |
| 对比分析 | `/task/compare` | 多任务指标对比 |
| 算力资源监控 | `/task/resourceMonitor` | 服务器汇总与列表 |
| 服务器详情 | `/task/resourceMonitor/detail/:serverIp` | 硬件信息、资源趋势、运行/排队任务 |

### 3.3 算力资源监控（重点模块）

该模块仅 **超级管理员**（`super_admin`）可访问，核心能力包括：

- 集群汇总统计（服务器总数、在线数、运行/排队任务数、平均 GPU 使用率）；
- 服务器纳管（手动添加 / 删除，有运行中任务时禁止删除）；
- 单台服务器资源详情与历史趋势图；
- 排队任务人工上下移动（仅改排序，不改优先级标签）；
- 取消排队。

**接口与业务规则详见：**

`src/pages/task/resourceMonitor/资源监视器API.md`

## 5. 系统架构

### 5.1 分层结构

```
┌─────────────────────────────────────────┐
│  Page 层（src/pages/）                  │  页面 UI、交互、路由
├─────────────────────────────────────────┤
│  Services 层（src/services/）           │  API 封装、统一请求
├─────────────────────────────────────────┤
│  Constants / Utils / Components       │  常量、工具、公共组件
├─────────────────────────────────────────┤
│  后端 API（/api/*）  │  MLflow（/mlflow-api/*）│
└─────────────────────────────────────────┘
```

### 5.2 请求约定

- **Base URL**：`/api`（在 `src/app.tsx` 的 `request` 配置中设定）；
- **认证方式**：`Authorization: Bearer {token}`，Token 存于本地 Storage；
- **响应格式**（二选一，前端均已兼容）：
  - `{ code: 200, data, message }`
  - `{ success: true, data, errorMessage }`
- **超时**：默认 10s（可在 `src/app.tsx` 调整）。

### 5.3 Mock 与真实接口

部分模块采用 **「优先请求真实接口，失败时 fallback Mock」** 策略，便于后端未就绪时演示：

| 模块 | Service 文件 | Mock 数据 |
|------|--------------|-----------|
| 算力资源监控 | `src/services/resourceMonitor.ts` | `src/pages/task/resourceMonitor/mockData.js` |
| 任务/模型/数据集等 | 各 service 文件 | `src/constants/mockData.ts`（页面内 fallback） |

后端联调完成后，只要接口可达，前端将自动使用真实数据，**无需修改页面代码**。

### 5.4 主要 API 端点（摘要）

完整端点定义见 `src/constants/platform.ts` 中 `API_CONFIG.ENDPOINTS`：

| 业务 | 路径前缀示例 |
|------|----------------|
| 模型 | `/model/list`、`/model/upload/*` |
| 数据集 | `/dataset/list`、`/dataset/upload/*` |
| 训练任务 | `/task/list`、`/task/create`、`/task/detail` |
| 算力监控 | `/resource-monitor/summary`、`/resource-monitor/servers` |
| MLflow 指标 | `/mlflow-api/2.0/mlflow/metrics/*`（经代理转发） |

## 8. 构建与生产部署

### 8.1 构建

```bash
npm install
npm run build
```

构建产物位于 `dist/` 目录，文件名含 hash，支持增量发布与缓存刷新。

### 8.2 路由模式

生产环境默认使用 **Hash 路由**（`.env.production` 中 `REACT_APP_HISTORY_MODE=hash`）。

- 优点：无需服务端额外配置 SPA 回退规则；
- 访问形式：`http://域名:端口/#/dashboard`

若需 Browser History 模式，将环境变量改为 `browser`，并在 Nginx 配置 `try_files` 回退至 `index.html`。

### 8.3 推荐部署架构

```
浏览器
  │
  ▼
静态服务器（Nginx / serve-dist.cjs，端口 8000）
  ├── /*          → dist/ 静态文件
  ├── /api/*      → 后端 127.0.0.1:8080
  └── /mlflow-api/* → MLflow 127.0.0.1:5000/ajax-api
```

**说明**：浏览器不能直接访问 `127.0.0.1:8080`（那是用户本机），必须由前端所在服务器做反向代理。

### 8.4 使用内置脚本一键启动

```bash
npm run serve:prod
```

默认监听 `0.0.0.0:8000`，可通过环境变量覆盖：

```bash
BACKEND_PROXY_TARGET=http://127.0.0.1:8080
MLFLOW_PROXY_TARGET=http://127.0.0.1:5000/ajax-api
HOST=0.0.0.0
PORT=8000
npm run serve:prod
```

更详细的部署说明见项目根目录 `DEPLOY_SERVER.md`。

### 8.5 Nginx 配置参考（可选）

若甲方使用 Nginx 托管，可参考：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/myapp-simple/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /mlflow-api/ {
        proxy_pass http://127.0.0.1:5000/ajax-api/;
        proxy_set_header Host $host;
    }
}
```

## 11. 运维与故障排查

### 11.1 常见问题

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 页面空白 | 静态资源路径错误 | 检查 `publicPath`、Nginx `root` |
| 接口 404 | 代理未配置 | 确认 `/api` 转发到后端 |
| 登录后跳回登录页 | Token 失效或未携带 | 检查后端鉴权、Storage 中 token |
| 算力监控无数据 | 后端未实现或不可达 | 开发期会自动 fallback Mock；生产需检查 `/resource-monitor` 接口 |
| 训练指标无曲线 | MLflow 代理失败 | 检查 `/mlflow-api` 转发与 runId |
| 构建失败 | Node 版本过低 | 升级至 Node >= 20 |

### 11.2 日志与监控建议

- 前端本身无独立日志服务，建议甲方在 **Nginx / 网关层** 记录访问日志；
- 接口异常由 `src/app.tsx` 统一弹窗提示，可在浏览器开发者工具 Network 面板排查；
- 生产环境建议配置 HTTPS 与 CSP 安全策略。

### 11.3 浏览器兼容

- 支持现代浏览器：Chrome、Edge、Firefox、Safari 最新两个大版本；
- 不支持 IE11（依赖 React 19 与现代 ES 特性）。

## 12. 安全说明

1. **认证**：所有业务请求需携带 Bearer Token，401 时自动跳转登录页；
2. **权限**：前端路由与按钮级权限仅作展示控制，**核心鉴权必须由后端强制执行**；
3. **敏感配置**：生产环境服务器地址、密钥等不应写入前端代码，应通过部署环境变量或网关配置；
4. **上传**：大文件采用分片上传，具体限制见 `UPLOAD_CONFIG`。
