## 七、对接异地后端服务器步骤（详细）

> 目标：让登录/注册/系统管理模块从“本地 mock / demo 接口”切换为“真实异地后端”。  
> 接口契约参考：`docs/后端API接口契约.md`。

### 7.1 后端准备（必须项）

- **开启 CORS**（若前端与后端不同域名/端口）：
  - **允许来源**：前端开发地址（如 `http://localhost:8000`）
  - **允许方法**：GET/POST/PUT/DELETE
  - **允许请求头**：`Content-Type`、`Authorization`
  - 若使用 Cookie：需 `Access-Control-Allow-Credentials: true`，并在前端 request 开启 `credentials: include`（当前前端未使用 Cookie 方式）

- **先打通登录闭环（建议先做这两个接口）**：
  - `POST /api/login/account`：登录成功返回 token（推荐统一 `code/msg/data`）
  - `GET /api/currentUser`：必须返回 `role`（`super_admin | normal_admin | user`），用于前端 `useAccess` 权限判断

### 7.2 前端请求如何指向异地后端（2 选 1）

#### 方案 A：后端开启 CORS（推荐）

- 前端保持请求为 `/api/...`
- 将 `src/app.tsx` 中 `request.baseURL` 指向异地后端域名（例如 `https://api.xxx.com`）

#### 方案 B：本地开发代理（后端未开 CORS 时推荐）

- 配置 `config/proxy.ts`，把 `/api/` 代理到异地后端：
  - `target: 'https://api.xxx.com'`
  - `changeOrigin: true`
- 开发时仍请求 `/api/**`，由本地 dev server 代发到异地后端

> 注意：proxy 仅对“本地开发服务”有效，生产环境需要 CORS 或同域部署。

### 7.3 关闭 system 模块 mock（关键）

当前这些文件会在开发环境默认走 mock（`process.env.NODE_ENV === 'development'`），导致不会请求真实后端：

- `src/services/system/user.ts`
- `src/services/system/log.ts`
- `src/services/system/role.ts`
- `src/services/system/permission.ts`

联调异地后端时需要：

- **临时做法**：把上述文件的 `USE_MOCK...` 改为 `false`（仅联调期间）
- **推荐做法**：改为环境变量控制（如 `REACT_APP_USE_MOCK=true/false`），实现一键切换 mock/真实后端

### 7.4 Token 存储 + Authorization 请求头（关键）

要让系统管理接口正常鉴权，必须做两件事：

- **登录成功后存 token**（例如 localStorage）
- **请求拦截器统一注入 Authorization**
  - `Authorization: Bearer <token>`

并建议实现：

- **401 统一处理**：清 token 并跳转 `/user/login`，提示“登录已过期”

> 当前登录页仍按 demo 的 `msg.status === 'ok'` 判成功；如果后端返回 `code/msg/data+token`，需要同步调整登录页的成功判断与类型定义。

### 7.5 联调顺序（推荐）

按以下顺序联调，最快定位问题：

1. **登录**
   - 登录成功 → 本地能拿到 token（浏览器 Application/LocalStorage 可看到）
2. **当前用户**
   - 刷新页面触发 `GET /api/currentUser`
   - 必须返回 `role`，且超管返回 `super_admin`
3. **系统管理-用户管理列表**
   - 打开 `/system/user`，确认请求打到真实后端：`POST /api/system/user/list`
4. **系统管理-日志管理列表**
   - 打开 `/system/log`，确认请求：`GET /api/system/log/list`
5. **角色/权限管理（仅超管）**
   - `/system/role`：`POST /api/system/role/list`
   - `/system/permission`：`GET /api/system/permission/tree`、`GET /api/system/permission/role/{roleId}`

### 7.6 联调验收清单（可复制给后端）

- **登录/鉴权**
  - [ ] 登录接口返回 token（或等价凭证）
  - [ ] 携带 `Authorization: Bearer token` 可访问系统管理接口
  - [ ] token 过期返回 401

- **角色权限**
  - [ ] `GET /api/currentUser` 返回 `role` 且枚举值固定：`super_admin | normal_admin | user`
  - [ ] 普通管理员：用户列表只能看到普通用户；日志列表只能看到普通用户日志且不包含管理员姓名/IP
  - [ ] 普通用户：访问 `/system/*` 返回 403

- **接口一致性**
  - [ ] 字段与样例对齐（详见 `docs/后端API接口契约.md`）
