# 系统管理模块 - 使用说明

## 一、角色与权限概览

| 角色         | 说明           | 系统管理菜单可见项           | 用户管理       | 角色/权限管理 | 日志管理     | 个人中心-我的操作记录 |
|--------------|----------------|------------------------------|----------------|---------------|--------------|------------------------|
| 超级管理员   | 唯一，最高权限 | 用户/角色/权限/日志         | 全部操作       | ✅            | 全部+导出+IP| ✅                     |
| 普通管理员   | 可多个         | 用户管理、日志管理           | 仅普通用户，无删除/导出 | ❌ 菜单隐藏 | 仅普通用户日志，无IP无导出 | ✅                     |
| 普通用户     | 无系统管理权限 | 无                           | 无访问         | 无访问        | 无访问       | ✅ 仅看自己的记录      |

- **无权限操作**：统一提示「无权限执行该操作」。
- **无权限页面**：直接访问 URL 会跳转 403 页「无权限访问该页面」。
- **敏感操作**：超管删除/禁用管理员账号、删除角色等会二次确认。

## 二、如何切换角色（开发环境 Mock）

当前通过 **本地存储** 模拟“当前登录用户角色”，用于验证不同权限。

### 1. 在控制台切换角色

打开浏览器控制台（F12），执行：

```javascript
// 切换为超级管理员（默认）
localStorage.setItem('mock_role', 'super_admin');

// 切换为普通管理员
localStorage.setItem('mock_role', 'normal_admin');

// 切换为普通用户
localStorage.setItem('mock_role', 'user');
```

执行后 **刷新页面**（F5），左侧菜单与各页能力会按新角色生效。

### 2. 无登录接口时的默认行为

若未对接登录接口（`/api/currentUser` 失败或未实现），在开发环境下会自动使用 mock 用户：

- 未设置 `mock_role` 时，默认使用 **超级管理员**。
- 设置 `mock_role` 后，使用对应角色（见上）。

Mock 用户对应关系：

- `super_admin` → 名称：admin（超管），userid：admin  
- `normal_admin` → 名称：manager1（普管），userid：manager1  
- `user` → 名称：test01（普通用户），userid：test01  

「我的操作记录」按 `userid` 过滤日志，因此切换为 `user` 后应能看到 test01 的操作记录。

## 三、验证权限的检查清单

### 超级管理员（mock_role = super_admin）

- [ ] 左侧出现「系统管理」→ 用户管理、角色管理、权限管理、日志管理。
- [ ] 用户管理：能看到所有用户（含超管、普通管理员、普通用户）；有角色筛选；可新增/编辑/禁用/删除/导出。
- [ ] 角色管理、权限管理：可正常打开并操作。
- [ ] 日志管理：系统日志 / 操作日志 Tab；列表含「操作人」「IP 地址」；有「导出 Excel」。
- [ ] 个人中心 → 我的操作记录：可查看（当前 mock 为 admin 的日志）。

### 普通管理员（mock_role = normal_admin）

- [ ] 左侧「系统管理」下只有「用户管理」「日志管理」（无角色管理、权限管理）。
- [ ] 用户管理：仅看到「普通用户」；无角色筛选；可新增/编辑/启用/禁用；无删除、无导出；尝试导出或删除时提示「无权限」。
- [ ] 新增/编辑用户时，角色下拉只有「普通用户」；若接口允许选管理员则前端会提示「无权限分配管理员角色」。
- [ ] 日志管理：仅普通用户操作日志；列表无「IP 地址」列；无「导出」按钮，手动触发导出会提示「无权限」。
- [ ] 直接访问 `/system/role` 或 `/system/permission` 会跳转 403。
- [ ] 个人中心 → 我的操作记录：可查看（当前 mock 为 manager1 的日志）。

### 普通用户（mock_role = user）

- [ ] 左侧无「系统管理」菜单。
- [ ] 直接访问 `/system/user`、`/system/role`、`/system/permission`、`/system/log` 均跳转 403。
- [ ] 个人中心 → 我的操作记录：仅能看到当前用户（test01）的操作记录；筛选项为时间/操作类型/操作内容/结果。

## 五、对接真实后端时

1. **登录与当前用户**  
   保证 `/api/currentUser`（或等价接口）返回字段中包含 **角色**，例如：  
   `role: 'super_admin' | 'normal_admin' | 'user'`，以便 `src/access.ts` 正确计算权限。

2. **关闭 Mock 角色**  
   去掉或不再依赖 `localStorage.mock_role`；若仍保留，建议仅限开发环境使用。

3. **用户列表/日志列表**  
   后端按当前用户角色过滤数据（普管只看普通用户、普通用户操作日志等），与前端 `currentUserRole`、`currentUsername` 等约定保持一致。

4. **接口权限**  
   删除、导出、分配管理员角色等接口应由后端再次校验角色，返回 403 或明确错误码，前端据此提示「无权限」。
